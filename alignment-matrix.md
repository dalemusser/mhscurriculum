# Standards ↔ Mission HydroSci Alignment Matrix

A simplified, two-direction view of how science-education standards line up with what is actually in Mission HydroSci. This document is the quick-reference; deeper detail lives in:

- [`standards/ngss.md`](standards/ngss.md) — verbatim NGSS Performance Expectations with all three dimensions (SEP, DCI, CC) for each cited and adjacent standard.
- [`standards/`](standards/README.md) — per-state alignment files (50 states + DC).
- [`docs/curriculum/`](docs/curriculum/) — the curricular source materials each MHS objective and activity is documented in.

The standards column uses **NGSS Performance Expectation codes** (`MS-ESS2-4`, etc.) as the canonical anchor. ~22 states + DC use these codes directly; the rest have state-specific codes that map to the same NGSS PEs (see per-state files in `standards/`). The MHS columns use MHS-internal objective codes (`W2.1`, `A3.1`, etc., from the Learning Progression) and unit titles from `mhsgrading/`.

> **Argumentation note:** MHS's full argumentation arc (`A1.1` → `A5.1` and the Unit 6 counter-critique) is a literal classroom implementation of **NGSS Science and Engineering Practice 7: Engaging in Argument from Evidence** at the grade 6–8 progression. SEP 7 cuts across every unit and is shown as its own row at the bottom of Table 1 rather than repeated against every PE.

## How to use this matrix

- **For dashboard messaging** — pair an MHS objective code shown in the dashboard with the corresponding row in Table 2 to surface the standard the student is making progress on.
- **For LLM evaluation of student work** — use the standard's full PE statement (in `standards/ngss.md`) plus its three NGSS dimensions to ground the evaluation prompt; use Table 2 to know which standard a given activity is supposed to be demonstrating.
- **For supplemental-activity generation** — use Table 2 to know which standard an under-performing student needs additional practice on, then mine `docs/curriculum/` for the alternative-conceptions / target-ideas content tied to that objective.
- **For deployment in a specific state** — check the per-state file in `standards/` for the local code translation; the alignment in this matrix carries over directly for any NGSS or NGSS-equivalent state, and through a documented crosswalk for the others.
